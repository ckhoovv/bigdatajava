데이터
-------------------------------------------------
데이터 1개 - 변수

데이터 여러개 - 배열, 클래스(dto), 컬렉션

데이터 저장 - 파일

데이터 저장/ 너무 많음 - DB

데이터 너무 너무 많음 - 빅데이터


컬렉션(collection)
------------------------------------------------
변수 : 	하나의 데이터의 하나의 이름
	하나의 변수명은 반드시 타입을 앞에다 지정
	여러개의 데이터가 여러개의 이름
	여러개의 변수명에 타입을 모두 지정

배열 :	하나의 이름으로 여러개의 데이터를 사용
	타입 지정도 한번만!
	크기가 고정, 같은 타입의 데이터만 묶을 수 있다.

컬렉션 :	하나의 이름으로 어러개의 데이터를 사용
	타입 지정도 한번만!
	크기가 유동성, 다른 차입의 데이터도 묶을 수 있다.
	클래스(DTO)를 묶을 때 보통 사용
	Vector

가방안에 들어간 물건들: 순서x, 중복x
어벤져스 예매줄: 순서o, 
	먼저 줄선사람이 먼저 예매하고 그 줄에서 나감
====================================
컬렉션의 데이터 형태에 따른 분류

List 타입: 순서o, 순위, ArrayList

Set 타입: 중복x, HashSet
Queus 타입: FIFO(First Input First Output, 선입선출)
	   LinkedList

Map 타입: 키 + 값, HashMap

==============================
DB 검색의 결과는 레코드의 집합인 테이블
(Resultset객체)
--------------------------------
하나의 레코드는 하나의 DTO에 mapping
여러개의 레코드는 여러개의 DTO에 mapping
=> 결국, DTO
